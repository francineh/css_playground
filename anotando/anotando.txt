cascade e especividade>>>>>>>>>>>>>>>>>>>>>>>>>..

quando voce tem duas classes iguais e repete a regra, minha cor é vermelha no primeiro seletor e no segundo é verde, devido ao sistema de cascada, meu seletor a ser aplicado é o segundo, por que uma regra sobre escreveu a outra.
exemplo:
.titulo{
    color:red;

}

.titulo{
    color:green;
}

exemplo 2

.titulo-pagna{
    color:red;
    font-size: 5rem;
}

.titulo{
    color:green;
}

embora sao classes com nomes diferentes, eles funcionanm como se fosse a mesma, detalhe que a classe titulo-pagina tambem se encontra no html.


.titulo-pagna{
    color:red;
    font-size: 5rem;
    border: 5px solid red;
}

.titulo{
    color:green;
    border-style: dashed;
}

regra 1: tudo que aparecer depois no meu css vai ter prioridade.

exemplos de especificidades
.titulo.titulo-pagna{aqui eu estou somando meu seletor.} eu so vou obdecer as regras a seguir para todos os elementos que tiverem essas duas classes. 

regra 2> somar o peso de cada especificidades.

quando especificado oo "id", algumas regras css foram ignoradas, ou seja o metodo cascata nao se aplicou nesse caso. exemplo :

#titulo-principal{
    margin-left:5rem;
}
/*1 - 0 - 0 = este seletor vai ter peso 1000 por que vou selecionar ele pelo id/ pegou nao importantando a posição que esta no css */

#titulo-principal{
color:aquamarine;


}
/* cascata e sobre escrever uma regra*/
/* 0 - 2 - 0 = seletor por duas classes , é peso somado, 2 classes peso 20, 3 classes peso 30*/
.titulo.titulo-pagna{color:#b66983;
margin-left:3rem;} /*este seletor é mais especifico que os seletores abaixo/por que eu quero todos que tiverem essas duas classes*/
/* 0 - 1 - 0 = seletor por classe tem peso 10*/
.titulo-pagna{
    color:red;
    font-size: 5rem;
    border: 5px solid red;
}
/* 0 - 1 - 0 = seletor por classe tem peso 10*/
.titulo{
    color:green;
    border-style: dashed;
}
/* 0 - 0 -1> seletor por tag tem peso 1*/
h1{
    color: blue;
}

 AULA DISPLAY MODE

 as div tem por padrao um display bloco, isso signica que ele ocupa toda o bloco da minha pagina, um quadrado como um todo.   
em outras palavras ocupam todo o espaço disponivel. 

se eu mudar a minha div para um display:inline; ele vai ocupar o suficiente para englobar meu texto.

regras:

DISPLAY:BLOCK> ACEITA O width eo height( altura e largura),padding,margin,e border,funciona na vertical e na horizontal.
sempre vao quebrar uma linha entre um elemento e outro.

DISPLAY:inline> NAO ACEITA O width eo height( altura e largura)
padding,margin,e border,funciona so na horizontal.

mais utilizados  por padrao sao inline.
<span>
<strong>
<em>
<a>
nao aceita margin-top, e nem margin-left

( quando se coloca o display:inline-block, ele passa respeitar, altura, largura, os padding,margin,border e afins. )
ainda assim ele se exforça em pegar somente o que esta caracterizado como block.se é so uma palavra sera apenas uma palavra. nao tem o mesmo efeito como as div.
é um misto dos dois.
